# ドキュメント作成ワークフロー

ドキュメント作成時は以下の手順を順番に実行する。前の手順が完了するまで次に進まないこと。

## 1. 規則の確認

docs/README.md を読み、以下の規則を確認する。

- 命名（ファイル名・ディレクトリ名）
- 構成（テンプレート・ポイント）
- 配置（判断基準・フロー）
- 文体（敬体・常体、文末表現、漢字の開き）

**完了条件:** 規則を把握した状態で次へ進む

## 2. 見出し構造の検討

ドキュメントの骨格となる見出し構造を設計する。

- 概要から始め、詳細へ展開する流れを意識
- 読者が最も参照する情報を中心に配置
- 補足的な情報は後半に配置

**成果物:** 見出し一覧（箇条書きまたはツリー形式）

## 3. 見出し構造のレビュー

設計した構造の妥当性を検証する。

| 観点 | 確認内容 |
|------|----------|
| 論理的順序 | 前の見出しが次の見出しの前提になっているか |
| 重複排除 | 同じ内容が複数の見出しに分散していないか |
| 粒度の統一 | 見出しの抽象度が揃っているか |

**完了条件:** 上記3観点すべてを確認し、問題があれば手順2に戻って修正

## 4. 内容の書き出し

各見出しの内容を記述する。

**原則:**
- 簡潔に書く（冗長な説明を避ける）
- 表・箇条書き・コード例を活用
- 実務で参照しやすい形式を優先

**成果物:** ドキュメント全文

## 5. 全体レビュー

完成したドキュメントを俯瞰して検証する。

| 観点 | 確認内容 |
|------|----------|
| 断絶 | 見出し間で内容が途切れていないか |
| 流し読み | 斜め読みで要点が把握できるか |
| 一貫性 | 全体を通してテーマが維持されているか |

**完了条件:** 上記3観点すべてを確認し、問題があれば該当箇所を修正

## チェックリスト

```
[ ] 1. docs/README.md を読み規則を確認した
[ ] 2. 見出し構造を設計した
[ ] 3. 見出し構造をレビューした（論理的順序・重複排除・粒度統一）
[ ] 4. 内容を書き出した
[ ] 5. 全体をレビューした（断絶・流し読み・一貫性）
```
