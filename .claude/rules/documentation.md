# ドキュメント作成ルール

## 作成手順

各手順は順番に実行し、前の手順が完了するまで次に進まないこと。

### 1. 見出し構造の検討

ドキュメントの骨格となる見出し構造を設計する。

- 概要から始め、詳細へ展開する流れを意識
- 読者が最も参照する情報を中心に配置
- 補足的な情報は後半に配置

**成果物:** 見出し一覧（箇条書きまたはツリー形式）

### 2. 見出し構造のレビュー

設計した構造の妥当性を検証する。

| 観点 | 確認内容 |
|------|----------|
| 論理的順序 | 前の見出しが次の見出しの前提になっているか |
| 重複排除 | 同じ内容が複数の見出しに分散していないか |
| 粒度の統一 | 見出しの抽象度が揃っているか |

**完了条件:** 上記3観点すべてを確認し、問題があれば手順1に戻って修正

### 3. 内容の書き出し

各見出しの内容を記述する。

**原則:**
- 簡潔に書く（冗長な説明を避ける）
- 表・箇条書き・コード例を活用
- 実務で参照しやすい形式を優先

**成果物:** ドキュメント全文

### 4. 全体レビュー

完成したドキュメントを俯瞰して検証する。

| 観点 | 確認内容 |
|------|----------|
| 断絶 | 見出し間で内容が途切れていないか |
| 流し読み | 斜め読みで要点が把握できるか |
| 一貫性 | 全体を通してテーマが維持されているか |

**完了条件:** 上記3観点すべてを確認し、問題があれば該当箇所を修正

## 手順チェックリスト

ドキュメント作成時は以下を順に実行する。

```
[ ] 1. 見出し構造を設計した
[ ] 2. 見出し構造をレビューした（論理的順序・重複排除・粒度統一）
[ ] 3. 内容を書き出した
[ ] 4. 全体をレビューした（断絶・流し読み・一貫性）
```

## ファイル命名規則

- `docs/`配下は小文字を使用
- 単語はハイフンで区切る

```
docs/
  versioning.md
  branching.md
  writing-guide.md
```

## 構成の基本形

```markdown
# タイトル

導入文（1-2行で目的を明示）

## 概要
## 本体セクション（複数可）
## 参考
```
