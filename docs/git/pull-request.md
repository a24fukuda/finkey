# Pull Request規約

本プロジェクトのPull Request（PR）の記述ルールを定める。

## PRタイトル

### 形式

```
<type>: <subject>
```

コミットメッセージのタイトルと同じ形式を使用する。PRに含まれる変更の主たる目的を表す。

### 例

```
feat: トレイメニューにアップデートボタンを追加
fix: オーバーレイのちらつきを修正
docs: CHANGELOGを追加
```

## PR本文

### 構成

| セクション | 内容 |
|-----------|------|
| Summary | 変更内容の概要（箇条書き1-3点） |
| Test plan | 動作確認の手順やチェック項目 |

### テンプレート

```markdown
## Summary
- 変更点1
- 変更点2

## Test plan
- [ ] 確認項目1
- [ ] 確認項目2
```

**記述のポイント:**
- Summaryは「何を」「なぜ」を簡潔に
- Test planはレビュアーが再現できる具体的な手順
