# バージョニングルール

本プロジェクトは[セマンティックバージョニング 2.0.0](https://semver.org/)に準拠する。

## バージョン形式

```
MAJOR.MINOR.PATCH
```

- **MAJOR**: 後方互換性のない変更
- **MINOR**: 後方互換性のある機能追加
- **PATCH**: 後方互換性のあるバグ修正

例: `1.9.0` → `1.10.0` → `2.0.0`

## バージョン番号の更新ルール

| 変更内容 | 更新対象 | 例 |
|---------|---------|-----|
| バグ修正（後方互換あり） | PATCH | 1.0.0 → 1.0.1 |
| 機能追加（後方互換あり） | MINOR | 1.0.1 → 1.1.0 |
| API非推奨化 | MINOR | 1.1.0 → 1.2.0 |
| 破壊的変更 | MAJOR | 1.2.0 → 2.0.0 |

**リセットルール:**
- MINOR更新時 → PATCHを0にリセット
- MAJOR更新時 → MINOR, PATCHを0にリセット

## 初期開発フェーズ（0.y.z）

`0.y.z`はAPIが不安定な初期開発段階を示す。

- いつでも破壊的変更が入る可能性がある
- 本番環境での利用は推奨しない
- `1.0.0`でパブリックAPIを確定

## プレリリース版

正式リリース前のテスト版には、ハイフン区切りで識別子を付与する。

```
1.0.0-alpha
1.0.0-alpha.1
1.0.0-beta.2
1.0.0-rc.1
```

**制約:**
- 使用可能文字: `0-9`, `A-Z`, `a-z`, `-`
- 数値のみの識別子は先頭ゼロ禁止

## ビルドメタデータ

ビルド情報はプラス記号で付与する。バージョン比較には影響しない。

```
1.0.0+build.123
1.0.0-beta+exp.sha.5114f85
```

## バージョン比較

優先度: MAJOR > MINOR > PATCH > プレリリース

```
1.0.0-alpha < 1.0.0-alpha.1 < 1.0.0-beta < 1.0.0 < 1.0.1 < 1.1.0 < 2.0.0
```

同一バージョンのプレリリースは通常版より優先度が低い。
